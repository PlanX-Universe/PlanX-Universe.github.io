Overview

PlanXFlow is a software development lifecycle designed for engineering AI planning systems. PlanXFlow provides a structured, waterfall-like yet iterative approach tailored to the unique challenges in AI planning.

The lifecycle encompasses ten phases, guiding researchers and practitioners from conceptualization and design to deployment, monitoring, and analysis. It acts as a foundational guide that helps establish a common language among stakeholders and improves traceability and reproducibility in AI planning projects.

The Ten Phases

  1. Requirements Analysis: Identify functional, non-functional, domain, and user requirements for the planning system.
  2. AI Planning Model Formulation: Define a suitable planning model based on planning type, world context, and user features.
  3. Domain Model Design: Formalize domain knowledge using planning languages such as PDDL, HPDL, or HDDL.
  4. System Design and Architecture: Design architecture and workflows that integrate planning functionalities and services.
  5. Planning Technology Selection: Evaluate and select existing planning tools before building new components.
  6. Implementation: Develop planning components, communication mechanisms, and supporting artifacts.
  7. Testing: Validate requirements with both isolated and integration testing across domains and workflows.
  8. Deployment: Install and configure components for reliable execution environments.
  9. Monitoring: Collect and process provenance data to improve quality, reproducibility, and explainability.
  10. Analysis: Analyze provenance and system behavior to drive iterative improvement.

Innovative Elements

  • Planning Model: Captures the interplay among planning type, world context, and user features.
  • Technology Selection: Emphasizes systematic selection of planning technologies.
  • Provenance Data: Supports traceability of data sources and transformations throughout the lifecycle.

Core Features

  • Adaptability: Tailor phases to specific project needs.
  • Scalability: Expand or condense lifecycle phases by project scale.
  • Transparency: Maintain clear and understandable process steps.
  • Future-Proof: Keep workflows easy to upgrade over time.
